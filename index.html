Projeto: IAM & AD Hardening

► RESUMO EXECUTIVO

Implementação de controles de segurança defensiva em ambientes corporativos Windows, integrando endurecimento de Active Directory com processos de Governança de Identidades (IGA). Objetivo: mitigar vetores críticos como Pass-the-Hash, movimentação lateral e escalada de privilégios até o controlador de domínio.

Análise de Impacto (BIA)

⚠ CENÁRIO SEM CONTROLES
  • Ransomware via SMBv1: propagação lateral descontrolada pode paralisar toda a produção. Em ambientes industriais, o custo de downtime pode ultrapassar R$100 mil/hora.
  • Contas Órfãs (LGPD Art. 6°): ex-colaboradores com acesso ativo violam o princípio de necessidade e abrem vetores de acesso privilegiado não monitorados.
  • Ausência de Tiered Model: comprometimento de uma estação comum (Tier 2) pode escalar diretamente para o controlador de domínio (Tier 0) via roubo de credenciais em memória.
  • NTLM/LLMNR habilitados: facilitam ataques de relay e poisoning em redes internas sem segmentação.
✓ VALOR DE NEGÓCIO COM CONTROLES
  • Contenção de incidentes: segmentação por tiers limita o blast radius — comprometimento de Tier 2 não alcança Tier 0.
  • Conformidade LGPD: processos automatizados de Leaver garantem desativação imediata com trilha de auditoria completa.
  • ROI mensurável: redução estimada de 80% no tempo de auditoria manual via automação PowerShell.
  • Postura proativa: logs avançados permitem detecção de Brute Force e Kerberoasting antes da escalada de privilégio.

Automação — Auditoria IAM

Invoke-IAMAudit.ps1 PowerShell 
<#
  Autor  : Vinicius Oliveira Lima
  Versão : 2.0  |  Requer: PS 5.1+, RSAT AD DS, Admin
#>
#Requires -RunAsAdministrator
#Requires -Modules ActiveDirectory
#Requires -Version 5.1

Set-StrictMode -Version Latest
$ErrorActionPreference = 'Stop'

param(
    [int]    $InactiveDays = 90,
    [string] $OutputFolder = 'C:\Audit_Reports',
    [switch] $PassThru
)

# Prepara diretório de saída
if (-not (Test-Path -Path $OutputFolder -PathType Container)) {
    New-Item -Path $OutputFolder -ItemType Directory -Force | Out-Null
}
$ReportPath = Join-Path $OutputFolder "Relatorio_IAM_$(Get-Date -Format 'yyyyMMdd_HHmm').txt"

try {
    # 1. PasswordNeverExpires via LDAPFilter (portável entre versões RSAT)
    #    FIX: -Filter com aspas simples NÃO expande variáveis PowerShell
    #    Bit 0x10000 de userAccountControl = ADS_UF_DONT_EXPIRE_PASSWD
    $PassNeverExpires = Get-ADUser `
        -LDAPFilter '(&(objectClass=user)(userAccountControl:1.2.840.113556.1.4.803:=65536)(!(userAccountControl:1.2.840.113556.1.4.803:=2)))' `
        -Properties PasswordLastSet |
        Select-Object Name, SamAccountName, PasswordLastSet

    # 2. Contas inativas usando lastLogonTimestamp (replicado entre DCs)
    #    FIX: LastLogonDate é alias local — no filtro AD usa-se FileTime
    $ThresholdFT = (Get-Date).AddDays(-$InactiveDays).ToFileTime()
    $InactiveUsers = Get-ADUser `
        -LDAPFilter "(&(objectClass=user)(!(userAccountControl:1.2.840.113556.1.4.803:=2))(lastLogonTimestamp<=$ThresholdFT))" `
        -Properties lastLogonTimestamp, Department |
        Select-Object Name, SamAccountName, Department,
            @{ Name='UltimoLogon'; Expression={
                if ($_.lastLogonTimestamp) {
                    [DateTime]::FromFileTime($_.lastLogonTimestamp)
                } else { 'Nunca' }
            }}

    # 3. Domain Admins com -Recursive
    #    FIX: sem -Recursive, grupos aninhados ficam invisíveis na auditoria
    $DomainAdmins = Get-ADGroupMember -Identity 'Domain Admins' -Recursive |
        Where-Object { $_.objectClass -eq 'user' } |
        Get-ADUser -Properties Title, LastLogonDate |
        Select-Object Name, SamAccountName, Title,
            @{ Name='UltimoLogon'; Expression={
                if ($_.LastLogonDate) { $_.LastLogonDate } else { 'Nunca' }
            }}

    # Monta relatório
    $sep = '=' * 53
    $R   = [System.Text.StringBuilder]::new()
    $null = $R.AppendLine("$sep`nRELATÓRIO DE AUDITORIA IAM`nGerado : $(Get-Date -Format 'dd/MM/yyyy HH:mm')`nAnalista: Vinicius Oliveira Lima`n$sep")
    $null = $R.AppendLine("`n[CRÍTICO T1078] CONTAS COM SENHA PERMANENTE ($($PassNeverExpires.Count))")
    $null = $R.AppendLine(($PassNeverExpires | Format-Table -AutoSize | Out-String))
    $null = $R.AppendLine("`n[MÉDIO T1078] CONTAS INATIVAS >$InactiveDays DIAS ($($InactiveUsers.Count))")
    $null = $R.AppendLine(($InactiveUsers | Format-Table -AutoSize | Out-String))
    $null = $R.AppendLine("`n[CONTROLE] DOMAIN ADMINS DIRETOS + ANINHADOS ($($DomainAdmins.Count))")
    $null = $R.AppendLine(($DomainAdmins | Format-Table -AutoSize | Out-String))

    Set-Content -Path $ReportPath -Value $R.ToString() -Encoding UTF8

    # Hash SHA-256 para integridade do relatório
    $Hash = (Get-FileHash -Path $ReportPath -Algorithm SHA256).Hash
    Add-Content -Path $ReportPath -Value "`nSHA-256: $Hash" -Encoding UTF8

    Write-Host "[OK] Relatório: $ReportPath" -ForegroundColor Green
    Write-Host "[OK] SHA-256  : $Hash"         -ForegroundColor DarkGreen
    if ($PassThru) { Get-Content $ReportPath | Write-Host }

} catch {
    Write-Error "Erro: $($_.Exception.Message)"
    exit 1
}

Hardening — Registro e Protocolos

Invoke-ADHardening.ps1 PowerShell 
# Desativar LLMNR — previne poisoning (T1557.001)
$llmnrPath = 'HKLM:\SOFTWARE\Policies\Microsoft\Windows NT\DNSClient'
New-Item         -Path $llmnrPath -Force | Out-Null
Set-ItemProperty -Path $llmnrPath -Name EnableMulticast -Value 0

# Desativar SMBv1 — previne EternalBlue / WannaCry
Set-SmbServerConfiguration -EnableSMB1Protocol $false -Force
Disable-WindowsOptionalFeature -Online -FeatureName SMB1Protocol -NoRestart

# Restringir NTLM — mitiga Pass-the-Hash (T1550.002)
# Valor 5 = NTLMv2 apenas; rejeita LM e NTLMv1
$lsaPath = 'HKLM:\SYSTEM\CurrentControlSet\Control\Lsa'
Set-ItemProperty -Path $lsaPath -Name LmCompatibilityLevel -Value 5

# Auditoria avançada de logon
auditpol /set /subcategory:"Logon"                           /success:enable /failure:enable
auditpol /set /subcategory:"Credential Validation"           /success:enable /failure:enable
auditpol /set /subcategory:"Kerberos Authentication Service" /failure:enable

IGA — Ciclo de Vida de Identidades

🟩 Joiner

PROVISIONING

Criação de conta com grupo mínimo necessário (RBAC). MFA obrigatório desde o primeiro acesso. Conta alocada na OU correta, sem privilégio administrativo padrão.

Least Privilege MFA

🟡 Mover

ACCESS REVIEW

Ao mudar de função, revisão imediata de grupos e acessos. Remoção de permissões anteriores — evita acúmulo de privilégios (privilege creep). Aprovação do gestor registrada.

Access Review SoD

🔴 Leaver

DEPROVISIONING

Desativação imediata ao desligamento (T0). Sessões encerradas, senha randomizada, conta movida para OU de quarentena. Exclusão após 30 dias com log auditável (LGPD Art. 6°).
